# EAMarketX

[eamarketx.com](https://eamarketx.com)

EAMarketX 是一个面向中文用户的 EA 交易市场，专注于自动化交易相关数字商品的发布、购买、下载和结算。

## 为什么是 EAMarketX

我们希望它看起来像一个正式、清晰、可信的产品，而不是临时拼起来的资源站。

- 支持 EA、MQL 文件、SET 参数、指标、脚本和交易工具
- 支持封面、截图、版本号、分类、价格和标签
- 支持购买后长期下载
- 支持收藏、评价和作者主页
- 支持平台托管结算与卖家提现管理

## 安全原理

EAMarketX 的交易流程不是“买家直接打款给卖家”，而是“平台先托管，再结算给卖家”。

这种方式的核心好处是：

1. 平台能自动记录订单与下载权限。
2. 平台能统一收取 5% 服务费。
3. 平台能保留退款、争议和风控处理能力。
4. 平台能把卖家结算与提现分离，减少混账。
5. 平台能让购买记录长期可查，买家不必担心原商品下架后找不到文件。

```ts
const platformFeeRate = 0.05;
const sellerPayoutRate = 0.95;
```

这意味着每笔成功交易都会按固定比例分账：

- 平台收取 5%
- 卖家结算 95%

## 自动化设计

平台把大量重复工作自动化了，包括：

- 订单检测
- 支付确认
- 购买权限发放
- 收藏和评价记录
- 卖家收益统计
- 24 小时后自动出金

这类设计适合高频交易场景，也更容易随着商品数量和用户数量增长而扩展。

## 去中心化思路

这里的“去中心化”不是区块链营销意义上的口号，而是产品结构上的分散控制：

- 商品由卖家自己发布和维护
- 文件由平台私有存储托管
- 购买记录独立保存
- 用户下载权限和商品展示分离
- 平台只管理规则、结算和安全边界

这样可以减少单点依赖，也让市场不会被单个商品状态变化影响整体体验。

## 交易与风控

平台侧会记录：

- 订单
- 购买权限
- 收藏
- 评价
- 提现申请
- 到账状态

这样做的目的，是把交易、结算和售后都留在系统内，不让流程散落到私聊、转账截图或手工对账里。
